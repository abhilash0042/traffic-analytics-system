ABSTRACT Aiming at the serious problems of missed detection, false detection and difficult deployment of
existing target detection algorithms when applied to traffic scenes, a vehicle detection model YOLOv8-FDD
with lower parameter count and higher accuracy is proposed in this paper. First, an integrated Feature
Sharing Detection Head is proposed to compress the amount of redundant parameters in the model head
while ensuring the accuracy of the algorithm. Second, to enhance the interaction between regression
features and classification features in the detection head, an Feature Dynamic Interaction Detection Head
is further designed based on the Feature Sharing Detection Head. Third, the Dilation-wise Residual (DWR)
module to strengthen the multi-scale feature extraction capability of the backbone network. Finally, the
dynamic upsampling module DySample is utilized to improve the nearest neighbor interpolation upsampling
in YOLOv8. Validation on the UA-DETRAC dataset shows that the proposed YOLOv8-FDD model’s
parameter count is only 72.89% of the original YOLOv8, with computational complexity slightly increasing
from 8.1 GFLOPs to 8.5 GFLOPs. The mAP50 and mAP50-95 improved by 0.7% and 1.3% respectively,
with FPS consistently above 300. Additionally, on a self-built traffic surveillance dataset, YOLOv8-FDD
outperformed the original algorithm. The error type statistics indicate that YOLOv8-FDD significantly
reduces both vehicle false negatives and false positives, making it a high-accuracy, well-generalized real-
time vehicle detection method.
INDEX TERMS Deep learning, vehicle detection, YOLOv8, DWRSeg, DySample.
I. INTRODUCTION
Vehicle detection is a subclass of object detection that uses
computer vision techniques to accurately identify the cat-
egory and location of target vehicles in given images or
videos. It is a crucial prerequisite for downstream tasks such
as vehicle tracking and traffic counting, and a fundamental
component of intelligent transportation systems [1]. Accu-
rate and fast vehicle detection technology plays a significant
role in enhancing traffic safety and improving traffic man-
agement. Currently, there are two main challenges when
applying object detection algorithms to traffic scenarios.
Firstly, ensuring the accuracy of vehicle detection algorithms
is essential. Traffic scenes present various complex environ-
mental factors, such as lighting changes, weather conditions,

the algorithm can ultimately be deployed on edge devices

predict the target category and location within these regions.

[3], [4], [5]. These algorithms have a more complex imple-
mentation process and slower detection speed, making
real-time traffic target detection difficult. To address these
issues, single-stage object detection algorithms have been
proposed. These algorithms use an end-to-end convolutional
neural network to directly predict the target’s category and
location information from the input image, compensating for
the lack of real-time performance in two-stage algorithms.
Common algorithms include the YOLO [6] series and SSD
[7] algorithms. Due to their speed advantage, single-stage
algorithms have been widely used for real-time vehicle detec-
tion [8]. The YOLO series of algorithms have gone through
iterations from early versions v2, v3, and v4 [9], and have
now evolved to YOLOv5, YOLOv6 [10], YOLOv7 [11],
YOLOv8, YOLOv9 [12], and YOLOv10 [13], proposed by
different teams. These algorithms continuously improve the
accuracy and speed of target detection. In recent years, the
YOLO series of algorithms have been continuously improved
for traffic detection tasks in order to meet the needs of the
development of intelligent transport systems.
Although new object detection algorithms are performing
increasingly well on large public datasets, in traffic scenar-
ios, factors such as vehicle occlusion and lighting variations
can lead to severe issues of vehicle missed detections and
false detections. To address these issues, researchers have
considered enhancing the vehicle features learned by the net-
work. For example, Song et al. [14] used MixUp and Mosaic
methods for data augmentation to strengthen the learning of
local vehicle features, and introduced the Efficient Channel
Attention (ECA) mechanism in the backbone network to
focus on useful features. They ultimately utilized a complex
decoupling head for prediction. Huang et al. [15] proposed
an improved DC-SPP-YOLO method based on Dense Con-
nectivity (DC) and Spatial Pyramid Pooling (SPP), which
strengthens feature propagation in the backbone network,
ensures the network retains more information flow, and intro-
duces an improved Spatial Pyramid Pooling (SPP) structure,
enabling the network to more comprehensively learn vehicle
features. Wang et al. [16] addressed the issue of difficult vehi-
cle feature capture on foggy days by improving the backbone
network using the Rep-ResNeXt reparameterization structure
and constructing a new Feature Enhancement Module (FEM)
that introduces an attention mechanism to help the network
focus on useful features in foggy scenes. They achieved good
detection results on the self-built Synscapes-Foggy dataset
and the open fog dataset RTTS.
The aforementioned studies have effectively enhanced
the accuracy of vehicle detection but have also increased the
complexity of the algorithms, making it difficult to ensure the
models remain lightweight. To meet the deployment needs
of traffic detection models, Alsanabani et al. [17] proposed
an enhanced YOLOv4-Tiny model. This model reduces the
floating-point operations in the backbone network while
maintaining the captured vehicle feature information, thereby
reducing the loss of accuracy caused by model lightweight-
ing. Experiments show that this model outperforms the
original in terms of inference speed and memory consump-
tion, while maintaining almost the same level of accuracy.
Feng and Yi [18] proposed a lightweight vehicle detection
network based on an improved YOLOv5. By combining
the detection head with the adaptive spatial feature fusion
block (ASFF head), multi-scale feature fusion is improved.
Extensive experiments on a small target vehicle dataset
demonstrate the superiority and lightweight nature of the
proposed method. Additionally, Guo and Zhang [19] used
the MobileNetV3 feature extraction network instead of the
YOLOv5 backbone network and further improved it with a
coordinate attention module, significantly reducing the num-
ber of model parameters.
Although the above studies have made some contributions
to the traffic detection task, there is still much room for
improvement in terms of model accuracy or the number of
parameters in the detection model, and there is no further
research on how the improved algorithm affects the leakage
and false detection situations. In this paper, we optimize the
efficient and lightweight YOLOv8n model to further improve
the model accuracy, reduce the number of model parameters,
and validate the impact of the improved model on vehicle
omission and false detection.
The main contributions of this study are as follows:
(i) Aiming at the problem that the amount of redundant
parameters in YOLOv8 detection head is too large, this
paper firstly designs a Feature Sharing Detection Head,
which compresses the number of model parameters and
computation amount while ensuring the lossless accu-
racy as much as possible, and improves the inference
speed of the model.
(ii) Considering that the detection head lacks feature inter-
action between the two tasks of classification and
localization, we further design the Feature Dynamic
Interaction Detection Head based on the Feature Shar-
ing Detection Head to enhance the classification and
localization ability of the model.
(iii) In order to enhance the network features, we improve
the feature extraction module for layers P4 and P5
of the backbone network based on the Dilation-wise
Residual (DWR) [20] module to enhance the informa-
tion captured from the extensible sensory field in the
deeper layers of the network.
(iv) Improve the nearest neighbor interpolation up-sampling
method in the original YOLOv8 using the dynamic
up-sampling operator DySample [21] to strengthen the
feature information after up-sampling.
(v) The improvement effect of the proposed algorithm
on the vehicle detection model is verified through
detection result visualization, receptive field visual-
ization, and error type analysis. Additionally, ablation
experiments on private traffic surveillance data with
significant variations in scenes and lighting are con-
ducted to further validate the generalization capability
of the proposed algorithm. Finally, we compare the
proposed algorithm with other popular object detection

II. LITERATURE REVIEW
The YOLO family of algorithms is nowadays widely used
in traffic control systems due to its high accuracy and
low cost. This section first briefly introduces the recent
developments and changes in the YOLO family of target
detection algorithms, and then analyzes the challenges and
their optimization methods when applying the YOLO family
of algorithms to real vehicle detection tasks.
Initially, the YOLO series algorithms were proposed
as single-stage object detection algorithms, successfully
addressing the issue of poor real-time performance in two-
stage algorithms. YOLOv4 [9] uses CSPDarknet53 as its
backbone network. This residual connection structure effec-
tively reduces the gradient vanishing problem by directly
feeding features to the output layer through branches and
also simplifies the model complexity. YOLOv5, developed
by Ultralytics, integrates five different model sizes: N, S,
M, L, and X. It uses CSPNet as its backbone network and
continuously improves its internal modules. YOLOv6 [10]
adopts an anchor-free design, simplifying the training and
inference process. YOLOv7 [11] introduces the Efficient
Layer Aggregation Network (E-ELAN), enhancing feature
representation capabilities while maintaining computational
efficiency. YOLOv8 optimizes the feature extraction module
of YOLOv5 and adopts the same decoupled head design as
YOLOX [22]. YOLOv9 [12] builds on YOLOv7 by intro-
ducing Programmable Gradient Information (PGI) and the
Generalized Efficient Layer Aggregation Network (GELAN)
architecture. The latest YOLOv10 [13], developed by the
Tsinghua University team, introduces a dual-label assignment
strategy and optimizes feature extraction and multi-scale fea-
ture fusion based on YOLOv8.
In the study of the latest YOLO-based methods, DEMO-
YOLO [23] designs a MAE-NAS network as the backbone
part of the traditional YOLO algorithm based on Neural
Architecture Search (NAS) and improves the Generalized-
FPN (GFPN) to propose an efficient RepGFPN as the
neck network to refine and fuse the high-level semantics
and low-level spatial features, design ZeroHead to better
balance the classification and regression tasks, and finally
improve the detection performance of small models through
knowledge distillation. Gold-YOLO [24] proposed a state-
of-the-art Gather-and-Distribute (GD) mechanism that injects
global information into higher-level networks through convo-
lution and self-attention operations, thus efficiently realizing
multi-scale information exchange, significantly enhancing
information fusion at the neck of the network, and achieving
a balance between latency and accuracy across all scale mod-
els. YOLO-World [25] is a real-time, open-vocabulary target
detection algorithm based on YOLOv8 by constructing a Pr-
parmeteriable Vision-Langeuate Path Aggregation Network
(RepVL-PAN) and Region-Text Contrast Loss to facilitate
the interaction between visual and linguistic information,
which reducing the dependence of traditional target detec-
tion algorithms on predefined object classes, and improves
the applicability of YOLO algorithms in open scenarios to
a certain extent. Mamba-YOLO [26] introduces and opti-
mizes a state space model (SSM), which effectively reduces
the computational burden of the secondary complexity of
the self-attention mechanism, and designs the LSBlock and
RGBlock modules to more accurately capture the local
dependencies of the image, which significantly enhances the
robustness of the model. TP-YOLO [27] introduces a con-
text converter and a full-dimensional dynamic convolution
module into the backbone network of YOLOv8 to enhance
its feature extraction capability, which is a lightweight
attention-based network that maintains the accuracy of the
target detection algorithm while significantly reducing the
model size and model inference time.
In summary, updates in the YOLO series algorithms mainly
focus on network structure optimization and training strategy
improvements. To address vehicle detection tasks in traffic
scenarios, specific improvements tailored to meet particular
requirements of the original algorithms are needed.
On the one hand, vehicle detection in traffic scenarios
is challenging due to the varying sizes of target vehicles
caused by different shooting distances and vehicle types.
Recognizing targets of different scales has become a major
difficulty. Chen et al. [28] first approached the problem by
segmenting images for detection using an adaptive cropping
algorithm and then merging the segmented detection results
with the global detection output. Zhang et al. [29] proposed
the RT-YOLO model, which improves the residual module
structure by enhancing the fusion of internal features within a
single layer, thus boosting the feature extraction capability of
the backbone network and improving the multi-scale feature
pyramid structure to enrich the neck features. Li et al. [30]
optimized the backbone feature network with bi-directional
convolution operations and constructed an adaptive loss func-
tion, resulting in the RES-YOLO model, which enhances the
detection performance of the traditional YOLO algorithm.
Padilla Carrasco et al. [31] considered replacing Focus layer
with multi-scale modules and adopted a bi-directional feature
pyramid network (Bi-FPN) to optimize multi-scale feature
fusion, though this approach increases computational cost.
In summary, aside from using data augmentation strategies,
mainstream methods to address the inconsistency of target
scales focus on enhancing the multi-scale feature extraction
capability of the backbone network and strengthening the
neck feature fusion, often at the expense of increased com-
putational load.

introducing additional parameters. The essence of attention
mechanisms is to assign weights to feature maps, enhancing
effective features and filtering out ineffective ones, as utilized
in the design of the Feature Dynamic Interaction Detection
Head in this paper.
In summary, to optimize object detection algorithms for
traffic target detection tasks, the key is to enhance the features
learned by the network while minimizing the cost increase
associated with improved accuracy. YOLOv8 continues
the decoupled detection head design from YOLOX, which
significantly increases the number of redundant parame-
ters for tasks like vehicle detection, where the number
of detection categories is relatively small. Therefore, the
YOLOv8-FDD detection model proposed in this paper starts
with the lightweight design of the detection head. It then
uses the DWR multi-branch dilated convolution structure to
strengthen the multi-scale features learned by the network.
Finally, it optimizes the upsampling module of the original
neck network to minimize the cost increase associated with
feature enhancement.
III. YOLOv8
YOLOv8 is a high-efficiency target detection algorithm intro-
duced by Ultralytics, which integrates the network structure
and modules of many mainstream target detection algo-
rithms, and has a strong potential for secondary development.
YOLOv8 officially provides five models of different sizes,
namely, YOLOv8n, YOLOv8s, YOLOv8m, YOLOv8l, and
YOLO8x, for users to use. They have the same network
structure, and the scale of the model is controlled by
two parameters: depth_multiple and width_multiple. The

The backbone of YOLOv8 network utilizes a CSP-
Darknet53 network structure consisting of multiple convo-
lutional modules, C2f modules, and an SPPF module for
expanding the receptive field. The feature information fed
from the backbone to the neck network consists of P3, P4, and
P5 feature maps, which are downsampled by factors of 8, 16,
and 32, respectively. As the sampling multiplicity increases,
the receptive field of the feature maps expands and contains
richer global information.
The neck of YOLOv8 network is used to fuse dif-
ferent scale feature maps from the backbone network,
so that the feature map input to the detection head con-
tains multi-scale feature information. It consists of two
feature fusion methods, FPN and PAN: FPN up-samples
the low-resolution feature maps and fuses them with
the medium-resolution feature maps from the backbone
part to obtain multi-scale feature information, while PAN
down-samples the high-resolution feature maps by using con-
volution to realize bottom-up feature fusion. The combination
of these two feature fusion methods forms a two-pathway
neck network structure, which helps to realize multi-scale
feature fusion perception.
The detection head of YOLOv8 uses a decoupled header
design similar to YOLOX, which implements the classifica-
tion task and the detection task using two separate branches,
where each branch consists of 2 convolutions with kernels for
3 and 1 convolution with kernels for 1. Since separate Detect
layers were used for the prediction of the P3, P4 and P5 layers
of the feature maps, this resulted in too many parameters in
the model header. To address this problem, two detection head
structures with low parameter quantities are designed in this
paper.
IV. YOLOv8-FDD
The YOLOv8n model has a good performance in terms of
vehicle detection accuracy and speed, especially in terms of
speed, which well meets the requirements of real-time vehicle
detection. In this paper, we try to improve the YOLOv8n
model by using the lightweight module structure to reduce
the number of model parameters while further improving
the model accuracy. Then, the features in the network are
strengthened by expanding the receptive field and optimizing
the up-sampling in order to improve the model accuracy,
which improves the vehicle leakage and false detection.
First, we designed a Feature Sharing Detection Head,
which dramatically compresses the number of model head
parameters by convolutional weight sharing and sharing one
feature extractor for different detection layers. Then, consid-
ering that the detection head lacks the interaction between
regression features and classification features, we make fur-
ther improvements based on the Feature Sharing Detection
Head and propose a Feature Dynamic Interaction Detection
Head to enhance the model’s classification and localization
capabilities. On this basis, we utilize the DWR module to
improve the feature extraction module in the deep layer of

A. FEATURE SHARING DETECTION HEAD
The YOLOV8 network head structure is designed using a
decoupled head, which means that the target classification
task and the bounding box regression task are handled sepa-
rately by two independent branches. While this approach will
enhance the feature extraction capability of the network to
some extent, it will also mean that each detection head needs
to use four convolutions with kernel 3 for feature extraction
and two convolutions with kernel 1 for regression predic-
tion and classification prediction. While YOLOv8 needs to
complete the detection of the feature layers of P3, P4, and
P5 scales respectively, which involves 12 convolutions with
kernel 3 and 6 convolutions with kernel 1, which greatly
increases the number of parameters, computation volume,
and inference time. The number of parameters in the head
of the YOLOv8n model accounts for 25% of the total num-
ber of parameters. Therefore, in this paper, we first address
the above problem by constructing a feature extractor using
two weight-sharing convolutions and use it to complete the
information extraction for three different scales of feature
layers. Based on this approach, the integrated Feature Sharing
Detection Head is proposed and its structure is shown in
Fig. 3.
FIGURE 3. Feature sharing detection head structure. The convolutions of
the same color share parameters, except for the 3 convolutions used to
adjust the number of channels, which do not share parameters.
The Feature Sharing Detection Head first adjusts the num-
ber of channels in the feature maps of the P3, P4, and P5
layers and feeds them into the same feature extractor to
learn features that are common to both the classification
and regression tasks. The feature extractor consists of two
parameter-sharing convolutions with kernel 3, which replaces
the 12 convolutions with kernel 3 in the original detection
head, effectively compressing the number of model param-
eters. Then, the classification and regression predictions are
accomplished by six convolutions with kernel 1, where three
classification prediction convolutions and three regression
prediction convolutions share weights, respectively. Finally,
to cope with the problem of inconsistent target scales detected
by each detection head, we use the Scale layer to adaptively
scale the feature maps in the regression branch. The scaling
layer is calculated as in (1).

C. DILATION-WISE RESIDUAL
Dilation-wise Residual (DWR) module from the DWRseg
[20] network is an efficient two-step residual feature extrac-
tion method (Regional Residualization-Semantic Residual-
ization), which can be used to enhance the feature information
extracted from scalable receptive fields from the higher levels
of the network. The structure of the DWR module is shown
in Fig. 6.
The module structure is designed using residual connec-
tivity, where a two-step method is used within the residuals
to efficiently learn multi-scale contextual information, and its
multi-branching structure is used to adapt to the requirements
of different sizes of receptive fields in one layer.
Step 1: Perform Region Residualization (RR) on the input
features. Specifically, the input features are subjected to a
combined operation of a 3∗3 convolution, Batch Normaliza-
tion and ReLU activation function to obtain region residual
features with different sizes for subsequent morphological
filtering.
Step 2: Semantic Residualization (SR). The regional resid-
ual feature maps generated in the first step are divided into
three branches, which are used to filter the regional features
morphologically by using the hollow depth convolution with
different null rates to extract the multi-scale semantic infor-
mation, and then, the feature maps are passed through the BN
layer in order to obtain the semantic residuals.
This ‘‘RR-SR’’ approach intelligently learns the feature
map based on the size of the receptive field in the second step
in order to reverse match the receptive field. We will utilize
the properties of the DWR module to enhance the multi-scale
feature extraction capability of the YOLOv8 deep backbone
network. Specifically, the DWR module is utilized to replace
Bottleneck in the C2f module at layers P4 and P5 in the
YOLOv8 backbone. The structure of the C2f module and the
C2f_DWR module is shown in Fig. 7.

D. DySample
The neck of the YOLOv8 network contains two Upsample
layers, which serve to expand the small-size feature maps in
the deeper layers of the backbone network in order to match
the feature map sizes from deeper to shallower layers. How-
ever, YOLOv8 employs the nearest neighbor interpolation
upsampling method, i.e., the pixel values of the nearest neigh-
bors are filled to enlarge the image size. The disadvantage
of this method is that the pixel changes are discontinuous,
resulting in the inability to observe the subtle changes in the
acquired image, and the sampled image produces jagged and
blurred performance. In this paper, we propose to replace the
Upsample layer with a dynamic upsampling method based
on point sampling (DySample) [21] to enhance the features
after upsampling to improve the feature fusion effect at the
neck of the network. The design of DySample avoids the
computationally expensive dynamic convolution, and instead
implements it with the essence of upsampling-point sam-
pling. DySample process is shown in Fig. 8.
DySample first uses a linear layer to map the input features
as point-by-point positional biases, which are then summed
with the original grid point coordinates to obtain new sam-
pling positions, and resampled at these new positions to
obtain the up-sampled features. In detail, since the sampled
points generated from the same point have the same initial
position, as in Fig. 9(a), this would ignore the mutual posi-
tional relationship of the up-sampled points. So, DySample
uses bilinear interpolation to initialize the sampled points,
separating the initial positions so that they are evenly dis-
tributed on the input feature map, as in Fig. 9(b). However,
without constraining the range of the four sampling point
offsets in Fig. 9(b), the offset ranges usually overlap, resulting
in prediction artifacts. Therefore, in Fig. 9(c), a range factor
is multiplied for the offsets to reduce the overlap. This solves
the problem of the initial positions of the sampling points and
their offset ranges.
In addition, in order to maintain a low computational
overhead while improving the performance of the module,
DySample adopts a group up-sampling approach, i.e., the
feature map is divided into four groups from the chan-
nel dimension, and the biases generated in each group are
FIGURE 9. Initial sampling positions and offset scopes.
up-sampled separately. In each grouping, features share the
same sampling set. The only additional computational cost
added by this is the linear mapping when generating biases,
which is negligible.

A. DATASET
1) UA-DETRAC DATASET
UA-DETRAC is a large-scale vehicle detection and tracking
dataset for real traffic scenarios produced by the University
at Albany. The dataset consists of 10 hours of video captured
by a Cannon EOS 550D camera at 24 different locations
in Beijing and Tianjin, China. The video was recorded at
25 frames per second and filmed at an angle close to the
surveillance camera view. A total of 8,250 vehicles and
1.21 million labeled bounding boxes of objects were man-
ually labeled. vehicles were divided into four categories, i.e.,
cars, buses, vans, and other vehicles. Weather conditions are
categorized into four types, i.e., cloudy, nighttime, sunny, and
rainy.
In the experiments of this paper, in order to avoid the
problem that the image changes between neighboring frames
are too small in UA-DETRAC, as well as the problem that
the shooting scenarios of the training set and the test set
are quite different, we have done the following treatment
to the UA-DETRAC dataset: the training set and the test
set are merged, and 10,870 images are screened from the
way of extracting one frame at certain frame intervals, and
re-divisioned into 8,564 training sets and 2,306 validation sets
at a ratio of 8:2, which is taken as the dataset used in this
paper.

C. EVALUATION INDICATORS
We use the number of parameters and the amount of com-
putation to evaluate the size of the model. The number
of parameters reflects the spatial complexity of the model,
and the amount of computation refers to the number of
floating-point operations per second of the model. The model
inference speed is reflected by the FPS indicator. It is worth
noting that when calculating the inference speed of small-
sized models, we ensure that the performance of hardware
devices, such as graphics cards, is fully released, and ulti-
mately calculate the FPS metrics based on the detection time
of each batch. This approach allows for a fairer and more
accurate comparison of the inference speeds of different sized
models. Precision, Recall, mAP50, and mAP50-95 are used
as evaluation metrics to assess the performance of the model.
Among them, the Precision and Recall are used as the basic
metrics, and the mean Average Precision calculated from
them is used as the final evaluation metric to measure the
recognition accuracy of the model. Their calculation formulas

VI. CONCLUSION
In this paper, we propose a target detection method based
on the improved YOLOv8n model for the vehicle detec-
tion task in traffic scenes. In the proposed method, firstly,
a Feature Sharing Detection Head is designed to address
the problem of large number of parameters in the head of
the YOLOv8 model, which maintains the accuracy basi-
cally unchanged while reducing the number of parameters
and computation significantly. Then, considering that both
the original YOLOv8 detection head and the Feature Shar-
ing Detection Head ignore the feature interaction between
the classification and regression tasks, we design a Feature
Dynamic Interaction Detection Head to enhance the classi-
fication and localization ability of the model (through the
Feature Dynamic Interactors). On this basis, we improve
the module of YOLOv8 backbone network deep layer by
utilizing the characteristics of DWR multi-scale feature cap-
ture, which makes the model computation and parameter
counts slightly reduced while the accuracy is improved.
Finally, the dynamic upsampling module DySample is used
to optimize the nearest neighbor interpolation upsampling
method in the original YOLOv8. We refer to the improved
model as YOLOv8-FDD and validated it on the filtered
UA-DETRAC dataset. The results show that the YOLOv8-
FDD model’s parameter count is only 72.89% of the original
YOLOv8, with computational complexity slightly increasing
from 8.1 GFLOPs to 8.5 GFLOPs. The mAP50 improved
from 97.1% to 97.8%, and the mAP50-95 increased from
83.6% to 84.9%. The detection speed remained consistently
above 300 fps, effectively meeting the requirements for
real-time vehicle detection.
Additionally, we conducted a deeper analysis of the
improvements in vehicle detection performance of YOLOv8-
FDD compared to the original YOLOv8n model (through
methods such as detection result visualization, receptive field
visualization, false detection type statistics, and generaliza-
tion performance verification on a private traffic surveillance
dataset). The results indicate that the proposed algorithm
effectively reduces both vehicle false negatives and false
positives. In the future, we will further optimize the network
model to enhance its accuracy and inference speed, enabling
better application in practical vehicle detection tasks.
